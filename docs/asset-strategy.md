# Asset Strategy

## Objetivo

Definir qué contenido visual o documental puede enviar el asesor según la etapa comercial, bajo qué condiciones y con qué límites.

## Principios generales

- Enviar solo assets relacionados con la propiedad activa.
- No repetir assets ya enviados salvo solicitud explícita del cliente.
- Priorizar calidad sobre cantidad.
- No enviar documentos vencidos.
- Registrar cada asset enviado para futuras decisiones.
- No enviar planes de pago sin vigencia confirmada.
- No enviar archivos internos de la empresa.
- No enviar listas completas de inventario al cliente.

## Catálogo permitido

```text
cover_image
property_gallery
exterior_gallery
interior_gallery
amenities_gallery
floor_plan
video
virtual_tour
brochure
payment_plan
price_list
location_map
reservation_requirements
```

## Estrategia por etapa

### Primera interacción

Permitido:
- respuesta textual;
- precio;
- disponibilidad;
- características principales;
- una imagen de portada.

No permitido:
- brochure completo sin necesidad;
- diez fotografías;
- plano;
- lista de precios completa;
- todos los documentos juntos.

Objetivo:
- responder lo esencial;
- no saturar al cliente;
- abrir espacio para profundizar si muestra interés.

### Interés en una propiedad

Permitido:
- galería de tres a cinco imágenes;
- características principales;
- ubicación general;
- amenidades relevantes.

Objetivo:
- ayudar a visualizar la propiedad;
- reforzar pertinencia;
- preparar paso a evaluación.

### Evaluación

Permitido:
- plano;
- video;
- recorrido virtual;
- comparación;
- plan de pago;
- lista de amenidades;
- información de entrega.

Objetivo:
- apoyar decisión seria;
- resolver objeciones;
- entregar contenido más profundo solo cuando el cliente ya está analizando.

### Intención alta

Permitido:
- brochure;
- requisitos de separación;
- requisitos de reserva;
- información de contacto;
- solicitud de visita;
- handoff humano.

Objetivo:
- facilitar la acción comercial siguiente;
- compartir documentos operativos solo cuando existe interés claro.

## Reglas específicas por tipo de asset

### `cover_image`
- se usa en primera interacción o en recomendaciones;
- debe ser una sola imagen principal.

### `property_gallery`
- se usa cuando ya existe interés concreto;
- preferir entre tres y cinco imágenes.

### `exterior_gallery`
- útil cuando la objeción o interés está relacionado con proyecto o fachada.

### `interior_gallery`
- útil cuando el cliente evalúa distribución o acabados.

### `amenities_gallery`
- usar cuando el cliente pregunta por amenidades o valor agregado del proyecto.

### `floor_plan`
- usar solo en `EVALUATION` o cuando el cliente lo pide explícitamente.

### `video`
- usar si existe activo confirmado y aporta claridad real.

### `virtual_tour`
- usar en `EVALUATION` o `HIGH_INTENT`, no como primer envío por defecto.

### `brochure`
- usar en `HIGH_INTENT` o cuando el cliente lo solicita explícitamente;
- evitar enviarlo de forma automática en los primeros turnos.

### `payment_plan`
- solo compartir con vigencia confirmada;
- si no hay confirmación, se deriva o se ofrece validación humana.

### `price_list`
- no compartir inventario completo por defecto;
- solo si existe una versión aprobada para clientes y el caso lo justifica.

### `location_map`
- usar cuando la ubicación es una objeción o criterio de decisión.

### `reservation_requirements`
- usar en intención alta o separación, no antes.

## Criterios de decisión

- Si el cliente hace una pregunta concreta, responder primero con texto y luego decidir si un asset aporta valor.
- Si no existe propiedad activa, no enviar assets específicos de una propiedad.
- Si el cliente cambia de propiedad, los nuevos assets deben corresponder solo a la nueva propiedad activa.
- Si el cliente ya recibió un brochure o un plan, no reenviarlo automáticamente en turnos posteriores.

## Relación entre etapa y profundidad

| Etapa | Profundidad recomendada |
| --- | --- |
| primera interacción | mínima y puntual |
| interés | visual y descriptiva |
| evaluación | comparativa y documental |
| intención alta | operativa y de cierre |
